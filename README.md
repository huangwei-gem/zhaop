# 飞书招聘人才库监控机器人

## 功能

### 1. 简历推送（自动）
- 定时检查人才库新增记录
- 当 AI 初筛结果不为空时，自动匹配面试官组合
- 按部门、岗位、城市匹配招聘任务中的面试官
- 推送含 **HR复核** 和 **业务复核** 按钮的评审卡片到对应群

### 2. HR/业务复核（卡片按钮）
- 点击「**通过**」或「**淘汰**」按钮
- 自动回填结果到人才库多维表
- 更新卡片状态（按钮置灰、显示已操作）

### 3. 一面评价
- 在群内发送「**姓名 一面评价：内容**」
- 自动写入人才库的「一面建议」字段
- 示例：郑杰 一面评价：沟通能力很强

### 4. 面试提醒
- 每天 **10:00** 和 **15:00** 自动触发
- @业务一面面试官 提醒关注招聘进展

### 5. 面试情况统计
- **定时推送**：每天 10:00 自动推送到各招聘群
- **手动查询**：群内 @机器人 发送「**面试情况**」
  
| 指标 | 说明 |
|------|------|
| 目标招聘人数 | 任务设置的招聘人数总和 |
| HR 推送数 | HR 复核结果 = 通过 |
| 面试中 | HR 和业务复核均通过，无一面评价 |
| 面试结束 | HR 和业务复核均通过，已有一面评价 |

### 6. 自动建群（新增）
- 每 10 分钟检查招聘任务表是否有新增任务
- 按面试官组合自动创建飞书群
- 群命名规则：**业务一面名字、HR二面名字、终面名字招聘群**
- 示例：黄维、帕合尔尼沙・阿不里孜、杜雁玲招聘群
- 建群时自动邀请所有面试官入群
- 群主设为业务一面面试官
- 跳过已存在的群，不重复创建

## 匹配逻辑

### 简历推送匹配
部门（二级 + 三级）、招聘岗位、招聘城市 **全部匹配** 才算匹配成功

## 使用说明

### 1. 简历推送
自动运行，无需手动操作。人才库新增 + AI 初筛有结果 = 自动推送

### 2. 评审卡片
点击卡片上的「**通过**」或「**淘汰**」按钮即可

### 3. 一面评价
在群中发送：郑杰 一面评价：沟通能力很强
格式：姓名 一面评价：内容

### 4. 面试提醒
自动触发，每天 10:00 和 15:00

### 5. 面试情况统计
- 每天 10:00 自动推送到各群
- 群内 @机器人 发送「**面试情况**」可手动触发

### 6. 自动建群
- 自动运行，每 10 分钟检查一次
- 只在招聘任务表有新增「招聘中」的任务时才会建群
- 不需要手动操作

## 飞书后台配置

### 必需权限
在飞书开发者后台 → 应用 → 权限管理 中开启：

| 权限 | 用途 |
|------|------|
| im:message | 发送消息 |
| im:chat | 创建群、加人 |
| bitable:app | 读写多维表 |
| contact:user.employee_id:readonly | 读取用户信息 |

### 可用范围
设置可用范围包含所有需要使用机器人的成员，否则建群时无法邀请不可见的用户

### 事件订阅
订阅方式选择「**长连接**」，注册事件：
- card.action.trigger（卡片回调）
- im.message.receive_v1（消息接收）

## 注意事项

- 首次启动会自动加载已有记录
- 关闭 CMD 窗口即可停止运行
- 建群时面试官需在机器人可用范围内，否则建群会失败
- 如需重启，先关闭当前窗口再双击 start.bat
- 启动后不要关闭 CMD 窗口，否则程序会停止
