## 功能概览

| 功能 | 触发方式 | 说明 |
|------|----------|------|
| 简历自动推送 | 自动（每 30 秒） | 人才库新增 + AI 初筛有结果，推送到对应面试官群 |
| HR/业务复核 | 点击卡片按钮 | 通过/淘汰，自动回填多维表，更新卡片状态 |
| 一面评价 | 群内发消息或点卡片 | 填写评价，自动写入人才库 |
| 面试提醒 | 自动（每天 10:00、15:00） | @业务一面面试官关注进展 |
| 面试统计 | 自动/手动 | 按面试官组合统计招聘进度 |
| 自动建群 | 自动（每 10 分钟） | 按面试官组合去重，自动建群拉人 |

## 飞书后台配置

### 第 1 步：创建应用

1. 打开 [飞书开发者后台](https://open.feishu.cn/app)
2. 点击 **创建企业自建应用**，填写名称、上传头像

### 第 2 步：获取凭证

左侧菜单 **凭证与基础信息** 页面，找到 **App ID** 和 **App Secret**，复制备用。

> 安全提醒：App Secret 相当于密码，不要泄露，不要上传到公开仓库

### 第 3 步：开启权限

左侧菜单 → **权限管理** → 开启以下权限：

| 权限 | 用途 |
|------|------|
| `im:message` | 发送消息到群聊 |
| `im:chat` | 创建群聊、邀请成员 |
| `bitable:app` | 读写多维表格 |
| `contact:user.employee_id:readonly` | 读取员工信息 |

### 第 4 步：配置事件

左侧菜单 → **事件与回调** → 订阅方式选择 **长连接**（WebSocket）

添加事件：
- `card.action.trigger` — 处理卡片按钮点击
- `im.message.receive_v1` — 接收群内消息

### 第 5 步：设置可用范围

左侧菜单 → **安全设置** → 设置可用范围，包含所有需要使用机器人的成员。

### 第 6 步：发布应用

点击右上角 **创建版本** → **申请发布**，审批通过后生效。

## 快速开始

### 1. 安装依赖

打开 CMD 运行：

```cmd
cd /d "C:\Users\35796\Downloads\飞书机器人"
npm install
```

安装成功后会生成 `node_modules` 文件夹。

### 2. 配置环境变量

打开 `start.bat`，修改以下两行（替换成你自己的）：

```batch
set APP_ID=你的AppID
set APP_SECRET=你的AppSecret
```

其他配置保持默认即可。

### 3. 启动机器人

**方式一**：直接双击 `start.bat`

**方式二**：CMD 中运行 `start.bat`

启动后出现 CMD 窗口，看到以下日志即成功：

```
[启动] Token 获取成功
[启动] 已加载 XX 条已有记录
[启动] 初始化完成
[启动] 长连接已启动
```

> 不要关闭这个窗口，关闭即停止机器人

## 功能使用手册

### 1. 简历自动推送

机器人每 30 秒检查一次人才库。当 **新增记录** 且 **AI 初筛结果不为空** 时，自动推送评审卡片到对应面试官群。无需手动操作。

### 2. HR / 业务复核

卡片上点击 **通过**（蓝色）或 **淘汰**（红色）按钮：
- 自动回填结果到人才库多维表
- 卡片更新为灰色，显示已操作

### 3. 一面评价

**快捷方式**：群内发送 `张三 一面评价：沟通能力很强`

**卡片方式**：群内发送 `一面评价` → 弹卡片选等级和原因

结果自动写入人才库的「一面建议」字段。

### 4. 面试提醒

每天 **10:00** 和 **15:00** 自动 @业务一面面试官。无需手动操作。

### 5. 面试统计

- **自动**：每天 10:00 推送到各招聘群
- **手动**：群内 @机器人 发送 `面试情况`

| 指标 | 含义 |
|------|------|
| 目标招聘人数 | 该组任务的人数总和 |
| HR 推送数 | HR 复核结果为通过 |
| 面试中 | 复核均通过，无一面评价 |
| 面试结束 | 复核均通过，已有一面评价 |

### 6. 自动建群

每 10 分钟扫描招聘任务表，对「招聘中」的任务按面试官组合去重，自动建群。

- 群名格式：`面试官1、面试官2招聘群`
- 自动邀请所有面试官入群
- 群主设为业务一面面试官
- 已存在的群不重复创建

> 面试官需在机器人可用范围内，否则拉人失败

## 匹配逻辑

人才库新增记录时，遍历所有任务计算得分：

- 二级部门匹配 +3 分
- 三级部门匹配 +2 分
- 招聘岗位匹配 +3 分
- 工作城市匹配 +2 分
- 状态为招聘中额外 +2 分

选最高分的任务作为匹配结果。

## 常见问题

**Q: Token 获取失败？**
检查 APP_ID 和 APP_SECRET 是否正确，应用是否已发布。

**Q: 建群提示未授权？**
需要管理员授权（首次运行时有提示），授权后 `user_token.json` 自动生成。

**Q: 卡片按钮没反应？**
检查飞书后台是否已添加 `card.action.trigger` 事件，订阅方式是否为长连接。

**Q: 如何停止？**
直接关闭运行中的 CMD 窗口。重启再次双击 start.bat。

**Q: 如何更新代码？**
CMD 运行 `git pull` 后再启动即可。

**Q: 重启后建群提示已存在？**
正常现象，已存在的群不会重复创建。
